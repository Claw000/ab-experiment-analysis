# AB 实验数据分析

## 适用场景

分析AB实验数据，给出效果判断和决策建议。数据来源可以是xlsx文件或直接查数据库。

## 指标体系

所有实验通用的指标框架。每个实验不一定分析所有指标，**有哪些就分析哪些**。

### 大盘体验指标
| 指标 | 单位 | 说明 |
|------|------|------|
| App次留 | % | 整体用户粘性 |
| App新用户次留 | % | 新用户留存 |

### 大盘收入指标
| 指标 | 单位 | 说明 |
|------|------|------|
| 人均充值 | 元 | 活跃用户平均充值 |
| 人均消费 | 元 | 活跃用户平均消费 |
| 总充值 | 元 | 充值总收入 |
| 总消费 | 元 | 消费总收入 |

### AI体验指标
| 指标 | 单位 | 说明 |
|------|------|------|
| AI渗透率 | % | 用户愿不愿意用AI |
| AI功能次留率 | % | 用完AI还回不回来 |
| **AI人均轮次** | 次 | **核心体验指标**，用得深不深 |
| AI提问率 | % | 多少用户主动提问 |
| AI提问次留率 | % | 提问用户次日回来继续问 |

### AI收入指标
| 指标 | 单位 | 说明 |
|------|------|------|
| 会员转化率 | % | 多少人开通会员 |
| 权益包转化率 | % / ‱ | 多少人购买权益包 |
| 会员收入 | 元 | 会员总金额 |
| 权益包收入 | 元 | 权益包总金额 |
| **会员+权益包收入** | 元 | **每次必算**，整体收入影响 |
| 会员ARPPU | 元 | 付费会员人均消费 |
| 权益包ARPPU | 元 | 权益包买家人均消费 |

**⚠️ 注意单位：** 权益包金额可能是"分"不是"元"，计算综合收入时先确认单位。

## 输入格式

### 方式1: xlsx文件（本技能默认）
- **单个xlsx**：一个指标一个文件
- **zip打包**：包含 `query_1.xlsx`、`query_2.xlsx` 等
- 典型列结构：`| pt(日期) | 分组(-1) | 分组(0) | 分组(1) | ... |`
  - 分组(-1)：大盘（仅参考）
  - 分组(0)：对照组
  - 分组(1/2/3...)：实验组

### 方式2: 直接读数据库
如果可以直连数据库，直接查询实验数据，跳过文件解压步骤。具体表名和字段以实际数据库为准，使用前先确认表是否存在。

## 分析流程

### Step 1: 读取数据
- xlsx方式：解压zip，读取每个文件，打印列名和前几行确认指标含义
- 数据库方式：按实验号查分组，关联指标表

### Step 2: 确定稳定期
- 看活跃用户数的进组曲线，判断流量何时稳定
- 稳定期判断：日增长 < 5% 开始算稳定
- 通常取实验开始后 **3-7天** 作为稳定期起点
- **从稳定期开始分析**，进组期数据不纳入

### Step 3: 进组均匀性检查
分析前先确认各组流量分配是否均匀：

```python
# 各组活跃用户数差异应 < 2%
for group in experiment_groups:
    ratio = active_users[group] / active_users[control]
    if abs(ratio - 1) > 0.02:
        print(f"⚠️ 分组{group}流量偏差 {(ratio-1)*100:+.1f}%")
```

如果分流不均（>2%），需要在结论中标注。

### Step 4: 计算核心统计量
对稳定期内的数据，每个指标计算：

```python
from scipy import stats
import numpy as np

mean_control = np.mean(control_data)
mean_experiment = np.mean(experiment_data)

diff = mean_experiment - mean_control
pct_diff = diff / mean_control * 100

t_stat, p_value = stats.ttest_ind(control_data, experiment_data)
significant = p_value < 0.05
```

### Step 5: 异常值检测
使用IQR法检测异常值：

```python
all_values = control + experiment
q1, q3 = np.percentile(all_values, 25), np.percentile(all_values, 75)
iqr = q3 - q1
lower, upper = q1 - 1.5 * iqr, q3 + 1.5 * iqr

outliers = [(date, value) for date, value in zip(dates, values) 
            if value < lower or value > upper]
```

- 发现异常值 → **呈现两版结果**（含异常值 + 去异常值）
- 去异常值时 **所有组统一去掉相同日期**
- 结论反转时重点标注 ⚠️

### Step 6: 综合收入计算（每次必做！）
**⚠️ 每次分析都必须计算会员+权益包总收入。**

```python
# 先确认单位！权益包可能是"分"
total_revenue = member_revenue + addon_revenue  # 单位一致时
total_revenue = member_revenue + addon_revenue / 100  # 权益包为分时
```

### Step 7: 趋势分析（判断"要不要继续跑"）
当结果不显著时，做趋势分析辅助决策：

```python
# 前后半段对比：差异在扩大还是收敛？
mid = stable_start + (len(data) - stable_start) // 2

first_half_diff = np.mean(exp_first) - np.mean(ctrl_first)
second_half_diff = np.mean(exp_second) - np.mean(ctrl_second)
```

**决策框架：**
- 差异在扩大 + p < 0.2 → 建议继续跑
- 差异在收敛 + p > 0.3 → 建议关掉
- 差异在收敛但用户想继续 → 尊重决策，给出预期

### Step 8: MDE分析（最小可检测效应）

```python
pooled_std = np.sqrt((np.std(ctrl)**2 + np.std(exp)**2) / 2)
n = len(ctrl)
mde = 2.8 * pooled_std / np.sqrt(n)  # 80% power, alpha=0.05
mde_pct = mde / np.mean(ctrl) * 100
```

> 当前样本可检测最小效应为 X%。如果真实效应 < X%，再跑多久都很难显著。

### Step 9: 分人群检查（整体无效时）
整体无显著差异时，看子群是否有效：
- 新用户 vs 老用户
- VIP vs 非VIP
- 不同渠道/平台

如果整体无效但某子群有效 → 说明干预方向对但目标人群不对。

### Step 10: 输出结论

**⚠️ 强制检查清单 — 输出前必须逐项确认，缺一不可：**

```
✅ 完成检查清单（按顺序执行，不能跳步）：
□ 稳定期计算 — 必须先算！看进组曲线，明确写出稳定期起止日期和天数
□ 进组均匀性检查 — 各组用户数差异 < 2%
□ 大盘体验指标（app次留、新用户次留）— 有数据就必须分析
□ 大盘收入指标（人均充值/消费、总充值/消费）— 有数据就必须分析
□ AI体验指标（渗透率、功能次留、人均轮次、提问率、提问次留）— 有数据就必须分析
□ AI收入指标（会员/权益包转化率、收入、ARPPU）— 有数据就必须分析
□ 综合收入（会员+权益包）— 每次必算！
□ 异常值检测 — 每次必做
□ 趋势分析（不显著时）— 前后半段对比
□ MDE分析（不显著时）— 最小可检测效应
```

**规则：**
1. **稳定期必须最先算**，所有指标分析都基于稳定期数据，不算稳定期后面全废
2. 数据里有的指标全部分析，不能跳过。没有的标注"无数据"

#### 输出格式

按四类指标分块呈现，**每一类都必须出现**（无数据的标注"本实验无此类数据"）：

```
📊 大盘体验指标
| 指标 | 对照 | 实验 | 差异 | 显著? |
（或：本实验无此类数据）

📊 大盘收入指标
| 指标 | 对照 | 实验 | 差异 | 显著? |
（或：本实验无此类数据）

📊 AI体验指标
| 指标 | 对照 | 实验 | 差异 | 显著? |
（或：本实验无此类数据）

📊 AI收入指标
| 指标 | 对照 | 实验 | 差异 | 显著? |
（或：本实验无此类数据）

✅ 检查清单确认：[逐项打勾]

结论：...
```

#### 显著性标记
- ✅ p < 0.05 显著
- ❌ p ≥ 0.05 不显著

#### 综合建议
- 全部显著正向 → 推全
- 体验显著正 + 收入无显著差异 → **推全**（体验提升+收入不降=纯赚）
- 方向正向但不显著 → 继续跑，等样本量
- 方向负向但不显著 → 警惕，继续观察
- 显著负向 → 关掉
- 体验正+收入负 → 权衡，问用户优先级
- 全部无显著 → 做趋势+MDE分析，给建议和依据

## 关键注意事项

### 常见陷阱
- **单位不一致**：权益包"分" vs 会员"元"
- **异常大单**：单个用户大额消费拉高均值
- **进组期数据**：前几天不稳定，不要纳入
- **样本量太小**：日均付费人数 < 50 时，t检验可靠性低
- **多重比较**：多个实验组时，注意Bonferroni校正

### 呈现规范
- 用表格呈现，不要长段文字
- 按四类指标分块（大盘体验/大盘收入/AI体验/AI收入）
- 异常值检测结果单独说明
- 综合结论放最后，包含明确建议
