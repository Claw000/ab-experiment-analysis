# AB 实验数据分析

## 适用场景

用户发来AB实验数据（xlsx文件，可能是zip打包），需要分析实验效果并给出结论。

## 输入格式

### 数据文件
- **单个xlsx**：一个指标一个文件
- **zip打包**：包含 `query_1.xlsx`、`query_2.xlsx` 等多个指标文件
- 每个文件通常包含：日期列 + 各分组的指标值

### 典型列结构
```
| pt(日期) | 分组(-1) | 分组(0) | 分组(1) | 分组(2) | 分组(3) |
```
- **分组(-1)**：大盘（不参与实验对比，仅作参考）
- **分组(0)**：对照组
- **分组(1/2/3...)**：实验组

### 指标类型
文件名或列名可能不包含指标名称。**必须向用户确认每个文件对应什么指标。**

常见指标分类：

| 类别 | 指标 | 单位 | 说明 |
|------|------|------|------|
| 体验 | AI渗透率 | % | 用户愿不愿意用 |
| 体验 | **人均轮次/人均使用次数** | 次 | **核心体验指标**，用得深不深 |
| 体验 | 次日留存率 | % | 用完还回不回来 |
| 商业-权益包 | 权益包人均消费 | 分 | 付费用户愿意花多少 |
| 商业-权益包 | 权益包消费总金额 | 分 | 权益包整体收入 |
| 商业-权益包 | 权益包付费率 | % | 多少人愿意付费 |
| 商业-权益包 | 权益包开通人数 | 人 | 绝对付费人数 |
| 商业-会员 | 会员开通率 | % | 护栏指标 |
| 商业-会员 | 会员开通人数 | 人 | 绝对开通人数 |
| 商业-会员 | 会员总金额 | 元 | 会员收入 |
| **综合** | **综合收入（每次必算）** | 元 | **会员+权益包总收入（需注意单位转换）** |

**⚠️ 注意指标归属：**
- 人均消费、消费总金额 → **权益包**数据
- 日均总金额、日均开通人数 → 可能是**会员**也可能是**权益包**，需确认
- 如果文件有活跃用户数，可用于归一化计算ARPU

## 分析流程

### Step 1: 解压和读取数据
```python
# 解压zip文件
import zipfile, openpyxl

# 读取每个xlsx，打印前几行，识别指标和分组
```
- 使用 `scripts/ab_analysis.py` 中的函数
- 先打印所有文件的列名和前几行数据，确认指标含义

### Step 2: 确定稳定期
- 实验通常有 **进组期**（用户逐渐进入实验）和 **稳定期**（流量稳定后）
- 稳定期判断：看活跃用户数是否趋于稳定（日增长<5%）
- 通常取 **实验开始后7天+** 作为稳定期起点
- 如果不确定，问用户"稳定期从哪天开始算？"

### Step 3: 计算核心统计量
对稳定期内的数据，计算每个指标：

```python
from scipy import stats
import numpy as np

# 均值
mean_control = np.mean(control_data)
mean_experiment = np.mean(experiment_data)

# 差异
diff = mean_experiment - mean_control
pct_diff = diff / mean_control * 100

# t检验
t_stat, p_value = stats.ttest_ind(control_data, experiment_data)

# 显著性判断
significant = p_value < 0.05
```

### Step 4: 异常值检测
使用IQR法检测异常值：

```python
all_values = control + experiment  # 合并所有组数据
q1, q3 = np.percentile(all_values, 25), np.percentile(all_values, 75)
iqr = q3 - q1
lower, upper = q1 - 1.5 * iqr, q3 + 1.5 * iqr

# 标记异常值
outliers = [(date, value) for date, value in zip(dates, values) if value < lower or value > upper]
```

- 如果发现异常值，**必须同时呈现两版结果**（含异常值 + 去异常值）
- 去异常值时，**所有组统一去掉相同日期**（公平对比）
- 去异常值后如果结论反转，要重点标注 ⚠️

### Step 5: 归一化（如有活跃用户数据）
```python
# ARPU = 收入 / 活跃用户 * 10000（万人均）
arpu = [rev / active * 10000 for rev, active in zip(revenue, active_users)]
```

### Step 6: 综合收入计算（每次必做！）
**⚠️ 每次分析都必须计算会员+权益包总收入，即使用户没有单独要求。** 这是判断实验对整体收入影响的核心指标。

```python
# 注意单位！权益包通常是"分"，会员通常是"元"
# 先确认单位再计算
total_revenue = member_revenue + addon_revenue / 100  # 分→元

# 如果单位一致（都是元），直接相加
total_revenue = member_revenue + addon_revenue
```

### Step 7: 输出结论

#### 输出格式（Discord/群聊）

```markdown
**实验名称 分析（稳定期 X/X-X/X）**

| 指标 | 对照(0) | 实验(1) | 差异 | 显著? |
|------|---------|---------|------|-------|
| 渗透率 | 19.04% | 19.17% | +0.13pp | p=0.67 ❌ |
| 综合收入 | 5,494元 | 5,076元 | -7.6% | p=0.21 ❌ |

**结论：**
1. 方向判断...
2. 护栏检查...
3. 建议...
```

#### 结论模板

**方向判断：** 用箭头表示方向
- ↗ 正向（实验 > 对照）
- ↘ 负向（实验 < 对照）
- → 持平（差异 < 1%）

**显著性标记：**
- ✅ p < 0.05 显著
- ❌ p ≥ 0.05 不显著

**护栏检查：** 护栏指标是否有显著负向（p<0.05）。**护栏指标因实验而异，必须向用户确认。** 常见护栏：会员开通率、次日留存、DAU、付费率等。

**综合建议模板：**
- 全部显著正向 → 建议扩流量/推全
- 方向正向但不显著 → 继续跑，等样本量
- 方向负向但不显著 → 警惕，继续观察
- 显著负向 → 建议关掉
- 体验正+收入负 → 需要权衡，问用户优先级

## 关键注意事项

### 1. 必须问的问题
- 这是什么实验？（实验内容/改动）
- 每个文件对应什么指标？（如果列名不明确）
- 各组分别做了什么改动？
- 稳定期从哪天开始？
- **这个实验的护栏指标是什么？**（不同实验护栏不同，不要默认是会员开通率）

### 2. 常见陷阱
- **单位不一致**：权益包金额可能是"分"，会员金额可能是"元"
- **异常大单**：单个用户大额消费拉高均值，必须检测
- **进组期数据**：前几天数据不稳定，不要纳入分析
- **样本量太小**：日均付费人数<50时，t检验可靠性低
- **多重比较**：多个实验组时，p值阈值可能需要Bonferroni校正

### 3. 呈现规范
- 用表格呈现，不要长段文字
- Discord/群聊不用markdown表格（用代码块或纯文本对齐）
- 每个指标标注"看什么"（体验/商业/护栏）
- 异常值检测结果单独说明
- 综合结论放最后，包含明确建议

### 4. 指标归属分类
分析时按以下分类分别呈现：
- 🔶 权益包（加量包）指标
- 🔷 会员指标
- 🔸 综合指标（会员+权益包）
